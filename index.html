Saara Ikonen -

Olen visuaalisen alan ammattilainen ja pian valmistuva IT-tradenomi.

Tälle portfoliosivulle sivulle on koottu joukko työnäytteitäni: linkkejä julkaisualustoille ja GitHubiin.

Kehitystyössä vahvuuteni tulevat erityisesti esille Front-End puolen kehittäjänä. Kokemusta on myös FullStack-kehitystyöstä.

  • Node.js REST API demo

    FullStack kurssityö: REST API. Käyttää Mongoosea API:n tietokantakyselyissä. Front-end rakennettu käyttäen EJS-sivupohjia. Renderissä julkaistussa versiossa voi tarkastella GET-metodia.

    Julkaistu Renderissä / GitHub

  • REACT demo

    Ylläolevalle REST API -työlle rakennettu REACT Front-end. Renderissä julkaistussa versiossa voi tarkastella hakutoimintoja, saattaa vaatia CORS-lisäosan toimiakseen.

    Julkaistu Renderissä / GitHub

  • Node.js demo "Guestbook"

    Node.js:llä ja Expressillä luotu vieraskirjan demosivu. Käyttäjä voi lähettää sivustolla viestejä, JSON ja HTTP POST / AJAX -harjoitus.

    Julkaistu Renderissä / GitHub

  • Verkkokauppa

    Verkkokaupan mockup -sivu. HTML & CSS harjoitus.

    https://haiku-art.github.io

  • "Elokuvaoraakkeli"

    JavaScript-kurssilla toteutettu elokuvasovellus, joka hyödyntää Finnkinon API:a.

    Julkaistu Netlifyssä / GitHub
Digitaalisten palveluiden käytettävyys ja saavutettavuus kiinnostavat minua aidosti. Olen idearikas ja luova, minulla on hyvä silmä väreille, sommitelmille ja yksityiskohdille. UI & UX suunnittelun työkaluna käytän Figmaa, jossa luonnistuvat niin layoutit, flowt, komponentit kuin design systemitkin.

  • UX Design Portfolio

    Figmassa julkaistu UX design -portfolio. Kurssityö, jossa tutkin, kuinka Laurean intranetissä olevien opastemateriaalien käyttökokemusta voisi kehittää. Tutkimusmenetelminä ja työkaluina on hyödynnetty haastatteluja, ja niiden pohjalta toteutettuja persoonia sekä käyttäjäpolkua (user journey).

    UX design portfolio Figmassa

  • UI prototyyppi: kuntosalisovellus

    Mobiilisovelluskurssille tehty UI prototyyppi. Ryhmätyö, jossa vastasin visuaalisesta ilmeestä. Toteutimme sovelluksen Ionicia ja Angularia hyödyntäen tämän prototyypin pohjalta.

    Prototyyppi / GitHub

  • UI prototyyppi: Ravintolan varaussivu

    Käyttöliittymän suunnittelu digitaalisen palvelun mallinnus -kurssilla. Prototyyppi on suunniteltu ja toteutettu annettujen "asiakasvaatimusten" pohjalta, joita olivat mm:

    • Helppokäyttöinen, visuaalinen, näyttävä, nopea käyttää (vähän ladattavia näyttöjä!)
    • Varauksen yhteydessä valitaan päivä, saapumisaika, varauksen kesto sekä henkilömäärä.
    • Järjestelmän tulisi tarjota selkeästi näytölle kaupungissa olevat ravintolat, joissa on vapaata tilaa valittuna ajankohtana, sekä ne, jotka palvelu kattaa, mutta joissa ei ole tilaa.
    Wireframet / Mobiili / Selain
